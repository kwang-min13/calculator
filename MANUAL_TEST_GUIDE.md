# 공학용 계산기 수동 테스트 가이드

이 문서는 공학용 계산기 웹앱의 UI 기능을 수동으로 테스트하기 위한 가이드입니다. 모든 테스트는 브라우저에서 직접 수행해야 합니다.

## 시작하기

1. **개발 서버 실행**
   ```bash
   powershell -ExecutionPolicy Bypass -Command "npm run dev"
   ```

2. **브라우저 접속**
   - URL: `http://localhost:5173/calculator/`
   - 계산기 UI가 정상적으로 표시되는지 확인

## 1. 기본 계산 플로우 테스트

### 1.1 단순 사칙연산
- [ ] **테스트**: `7 + 3 =`
  - **예상 결과**: `10`
  - 히스토리에 `7 + 3` 표시
  - 결과에 `10` 표시

- [ ] **테스트**: `15 - 8 =`
  - **예상 결과**: `7`

- [ ] **테스트**: `6 × 7 =`
  - **예상 결과**: `42`

- [ ] **테스트**: `20 ÷ 4 =`
  - **예상 결과**: `5`

### 1.2 연산자 우선순위
- [ ] **테스트**: `2 + 3 × 4 =`
  - **예상 결과**: `14` (2 + 12)
  - 곱셈이 먼저 계산되어야 함

- [ ] **테스트**: `10 - 2 × 3 =`
  - **예상 결과**: `4` (10 - 6)

### 1.3 괄호 사용
- [ ] **테스트**: `(2 + 3) × 4 =`
  - **예상 결과**: `20`

- [ ] **테스트**: `((2 + 3) × 4) - 5 =`
  - **예상 결과**: `15`

### 1.4 소수점 계산
- [ ] **테스트**: `3.5 + 2.5 =`
  - **예상 결과**: `6`

- [ ] **테스트**: `0.1 + 0.2 =`
  - **예상 결과**: `0.3` (근사값)

### 1.5 AC (초기화) 버튼
- [ ] **테스트**: 숫자 입력 후 `AC` 클릭
  - **예상 결과**: 
    - 히스토리 비워짐
    - 결과가 `0`으로 초기화

## 2. 공학 함수 테스트

### 2.1 삼각함수 (DEG 모드)
- [ ] **테스트**: `sin(30) =`
  - **예상 결과**: `0.5` (근사값)
  - DEG 모드 확인

- [ ] **테스트**: `cos(60) =`
  - **예상 결과**: `0.5` (근사값)

- [ ] **테스트**: `tan(45) =`
  - **예상 결과**: `1` (근사값)

### 2.2 삼각함수 (RAD 모드)
- [ ] **모드 전환**: DEG → RAD 클릭
  - **예상 결과**: RAD 버튼이 활성화됨

- [ ] **테스트**: `sin(1.5708) =`
  - **예상 결과**: `1` (π/2 근사값)

- [ ] **테스트**: `cos(3.14159) =`
  - **예상 결과**: `-1` (π 근사값)

### 2.3 로그 함수
- [ ] **테스트**: `ln(2.718) =`
  - **예상 결과**: `1` (e 근사값)

- [ ] **테스트**: `log(100) =`
  - **예상 결과**: `2`

### 2.4 제곱근
- [ ] **테스트**: `sqrt(16) =`
  - **예상 결과**: `4`

- [ ] **테스트**: `sqrt(2) =`
  - **예상 결과**: `1.414...`

### 2.5 거듭제곱
- [ ] **테스트**: `2 ^ 3 =`
  - **예상 결과**: `8`

- [ ] **테스트**: `10 ^ 2 =`
  - **예상 결과**: `100`

### 2.6 중첩 함수
- [ ] **테스트**: `sin(cos(45)) =`
  - **예상 결과**: 계산 완료 (에러 없음)

- [ ] **테스트**: `sqrt(16 + 9) =`
  - **예상 결과**: `5` (sqrt(25))

## 3. 에러 처리 테스트

### 3.1 0으로 나누기
- [ ] **테스트**: `10 ÷ 0 =`
  - **예상 결과**: 
    - `Math Error` 메시지 표시
    - 빨간색으로 표시됨

### 3.2 잘못된 수식
- [ ] **테스트**: `(2 + 3` (괄호 불일치)
  - **예상 결과**: `Syntax Error` 메시지

### 3.3 에러 후 복구
- [ ] **테스트**: 에러 발생 후 `AC` 클릭
  - **예상 결과**: 
    - 에러 메시지 사라짐
    - 결과가 `0`으로 초기화
    - 정상 동작 재개

- [ ] **테스트**: 에러 발생 후 숫자 입력
  - **예상 결과**: 
    - 자동으로 초기화됨
    - 새로운 계산 시작

## 4. 키보드 입력 테스트

### 4.1 숫자 키
- [ ] **테스트**: 키보드로 `0-9` 입력
  - **예상 결과**: 각 숫자가 디스플레이에 표시됨

- [ ] **테스트**: 키보드로 `.` (소수점) 입력
  - **예상 결과**: 소수점이 입력됨

### 4.2 연산자 키
- [ ] **테스트**: `+` 키 입력
  - **예상 결과**: 덧셈 연산자 입력

- [ ] **테스트**: `-` 키 입력
  - **예상 결과**: 뺄셈 연산자 입력

- [ ] **테스트**: `*` 키 입력
  - **예상 결과**: 곱셈 연산자 입력

- [ ] **테스트**: `/` 키 입력
  - **예상 결과**: 나눗셈 연산자 입력

### 4.3 특수 키
- [ ] **테스트**: `Enter` 키
  - **예상 결과**: 계산 실행 (= 버튼과 동일)

- [ ] **테스트**: `Escape` 키
  - **예상 결과**: 초기화 (AC 버튼과 동일)

- [ ] **테스트**: `Backspace` 키
  - **예상 결과**: 마지막 문자 삭제

### 4.4 괄호 키
- [ ] **테스트**: `(` 키 입력
  - **예상 결과**: 여는 괄호 입력

- [ ] **테스트**: `)` 키 입력
  - **예상 결과**: 닫는 괄호 입력

## 5. 접근성 테스트

### 5.1 키보드 네비게이션
- [ ] **테스트**: `Tab` 키로 포커스 이동
  - **예상 결과**: 
    - 버튼들 사이로 포커스가 이동함
    - 포커스된 버튼에 시각적 표시 (outline 또는 ring)

- [ ] **테스트**: `Enter` 또는 `Space`로 버튼 활성화
  - **예상 결과**: 포커스된 버튼이 클릭됨

### 5.2 ARIA 속성 확인 (개발자 도구)
- [ ] **테스트**: 디스플레이 영역 검사
  - **확인 사항**: 
    - `role="region"`
    - `aria-label="Calculator display"`
    - `aria-live="polite"` (결과 요소)

- [ ] **테스트**: 버튼 검사
  - **확인 사항**: 각 버튼에 적절한 `aria-label`

- [ ] **테스트**: 모드 토글 검사
  - **확인 사항**: 
    - `role="radiogroup"`
    - 각 라디오 버튼에 `aria-label`

### 5.3 스크린 리더 (선택사항)
- [ ] **테스트**: NVDA 또는 VoiceOver로 테스트
  - **확인 사항**: 
    - 버튼 레이블이 올바르게 읽힘
    - 결과 변경 시 알림이 읽힘
